# Design

## Goal

Create or update `.agents/TaskLogs/Task.md` with the task-level understanding, constraints, affected areas, and acceptance criteria.

## Write Scope

Only update `.agents/TaskLogs/Task.md` unless the user explicitly asks for implementation.

## Required Sections

- `# !!!TASK!!!`
- `# PROBLEM DESCRIPTION`
- `# UPDATES`
- `# INSIGHTS AND REASONING`
- `# AFFECTED PROJECTS`
- `# ACCEPTANCE CRITERIA`
- `# KNOWLEDGE BASE IMPACT`
- `# !!!FINISHED!!!`

## Process

1. Read the source request or selected Scrum task.
2. Identify hidden constraints, affected modules, existing examples, and non-goals.
3. Record build/test/runtime targets in `# AFFECTED PROJECTS`.
4. Define acceptance criteria as checkboxes.
5. Note durable KB/Learning impact.
6. End with `# !!!FINISHED!!!`.
