# Review

## Goal

Review a durable artifact or code change for correctness, missing tests, regressions, and project-rule violations.

## Process

1. Identify the review target and dimensions.
2. Read relevant `.agents/Guidelines/`, `.agents/KnowledgeBase/`, and `.agents/Learning/`.
3. Record findings with severity, evidence, required action, and resolution status.
4. Distinguish required changes from justified pushback.
5. Update `.agents/TaskLogs/Review.md` when review state must persist.
