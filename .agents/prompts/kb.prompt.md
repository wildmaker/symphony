# Knowledge Base

## Goal

Create or improve stable KnowledgeBase entries from source evidence, specs, docs, and repeated task experience.

## Process

1. Identify whether the topic is API selection, design explanation, testing knowledge, or integration knowledge.
2. Use `.agents/KnowledgeBase/KB_TOPIC_TEMPLATE.md` or `KB_DESIGN_TOPIC_TEMPLATE.md`.
3. Anchor claims to source files, tests, specs, or official docs.
4. Write guidance as decisions: "Use X when Y."
5. Link related topics and update `.agents/KnowledgeBase/Index.md`.
